## 生成判断

social-media-skills 是中文社媒内容 skill library，没有传统 App UI。按产品封面处理时，主证据适合展示真实产物：小红书封面与笔记、抖音口播与分镜、公众号标题组与长文结构。

本次输出采用“内容生成系统预览”：顶部是主题输入，中部是三个等权产物窗口，底部总结同一系统下的三种平台语义。平台差异放在内容结构和细节中表达：小红书是封面/笔记，抖音是 9:16 竖屏视频 frame，公众号是标题组/长文结构。

PNG 使用 `rsvg-convert -w 1080 -h 1440` 从 SVG 导出，校验尺寸为 1080x1440。

## Prompt 原则

- `prompt.md` 保持真实用户请求，不写完整视觉规约。
- skill 和仓库上下文由隔离 Codex 自己读取。
- `notes.md` 记录 worker 实际读了什么、如何判断、哪里暴露问题。
- `output.svg` 是 deterministic SVG test，用内部形状和具体产物文本模拟“生成图工具产物”；它不是模型生图结果。

## 本轮根据用户反馈改了什么

- skill 色彩规则改成正向 taste：色彩从品牌、主题、素材、情绪目标推导，建立稳定视觉系统。
- 补充色彩职责：主色、中性色、强调色职责清晰，保证层级、对比、可读性；同一系列 case 可共享气质，同时按主题调整配色策略。
- 补充安全区硬指标：卡片内边距 >= 24px；主要文字离容器边界 >= 20px；底部内容保留安全区；导出前检查文字不出界、不贴边、不重叠。
- 补充跨平台内容包规则：窗口可以等权；平台差异通过内容、比例和细节表达。
- social-media-skills 图改为深色产品系统 + 纸质产物窗口，三平台等权。
- 抖音窗口保留竖屏视频语义：手机 frame、播放符号、字幕条、进度条、镜头序列。
- 公众号窗口增加内边距和底部安全区，文字区未贴底、未溢出。

## 哪些规则有效

- “无传统 UI 的开源产品，优先展示真实生成结果或 case 成品”有效。
- “产品封面结构：工具名 + 一句话定位 + 真实证据 + 底部卖点”有效。
- “跨平台内容包使用等权产物窗口”有效。
- “抖音/视频产物出现竖屏 frame、进度条、字幕或镜头序列”有效。
- “公众号标题组和长文结构需要充足内边距”有效。
