# 小红书封面

小红书 3:4 封面路由。归属 `xiaohongshu`，不是独立同层 skill。

## 先做封面

- 封面决定点击，反馈最快。
- 开源工具需要一句话定位，封面会逼出核心卖点。
- 封面可复用到图文第 1 页、仓库社媒预览、抖音封面。

## 输入

- 工具名。
- 一句话能力。
- 目标用户。
- 最大痛点。
- 1-3 个可证明亮点。
- 截图、logo、demo 图路径。

## 类型路由

先分型，再构图。不要所有封面都做成大字海报。

| 类型 | 适用内容 | 第一视觉 | 读 |
| --- | --- | --- | --- |
| 产品封面 | App、网站、CLI、插件、SaaS、开源工具 | 品牌名 + 真实截图 | `covers/product.md` |
| 方法封面 | 教程、流程、经验、Skill 方法 | 方法名 + 步骤感 | `covers/method.md` |
| 观点封面 | 判断、反常识、痛点、趋势 | 一句冲突观点 | `covers/opinion.md` |
| 资料封面 | 合集、清单、模板、案例库 | 范围 + 可拿走感 | `covers/resource.md` |
| 活动海报 | 招募、系列预告、详情长图、活动回顾 | 活动价值 + 下一步动作 | `covers/event.md` |

Kaji 属于产品封面。它不是“额度跳出来”，而是把 AI working 状态、成本、压力、模型使用收束到低打扰状态栏。

## 常用公式

- 痛点型：`还在手搓 X？`
- 结果型：`我把 X 变成了 Y`
- 对比型：`以前 X / 现在 Y`
- 清单型：`3 个步骤做出 X`
- 反常识型：`别先写正文，先做封面`

## 视觉默认

- 比例：1080 × 1440，3:4。
- 主标题：3-10 字，手机信息流可读。
- 副标题：不超过 20 字。
- 层级：主标题、副标题、角标三层以内。
- 工具类封面：大字承诺 + 真实截图/生成结果 + 1-3 个短卖点。
- 色彩从品牌、主题、素材、情绪目标推导；建立稳定视觉系统。
- 主色、中性色、强调色职责清晰；保证层级、对比、可读性。
- 同一系列 case 可共享气质，同时按主题调整配色策略。
- 卡片内边距 >= 24px；主要文字离容器边界 >= 20px；底部内容保留安全区。
- 导出前检查文字不出界、不贴边、不重叠。
- 纯代码截图只作辅助证据；不写未证实数字。

## 通用检查

- 主标题 1 秒可读。
- 承诺具体，不是泛泛“效率提升”。
- 视觉主体真实，不像装饰插画。
- 层级干净，三层信息以内。
- 色彩系统服务产品气质，主色、中性色、强调色分工稳定。
- 文字与内容容器安全区足够，底部信息不贴底。

低于 8 分，不进入 case。
