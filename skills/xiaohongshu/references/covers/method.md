# 方法封面

适用于教程、流程、经验总结、Skill 方法。方法封面不是产品发布图，重点是“照着做也能复用”。

## 结构

```text
顶部：方法名或结果承诺
中部：3-5 个步骤、流程箭头或示例结果
底部：适用人群 / 使用场景 / 可保存提示
```

## 原则

- 标题说结果，不说“干货分享”。
- 步骤少而具体，避免把正文提纲整页塞进封面。
- 可以做流程感、卡片感、清单感；不需要强行放产品截图。
- 数字必须真实。没有确定数量时，用“几个”“一套”“从 0 到 1”。
- 视觉要有收藏感，但不要堆贴纸、emoji、荧光笔。

## 检查

- 看完封面能知道方法解决什么问题。
- 步骤看起来能执行，不是概念词堆叠。
- 视觉有顺序感，不是散点信息。
- 保存价值明确。
