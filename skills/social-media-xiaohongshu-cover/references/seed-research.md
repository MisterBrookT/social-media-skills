# 小红书封面种子调研

调研日期：2026-07-09。

## 结论

`social-media-skills` 第一张小红书封面不应做成“代码仓库宣传图”。更有效的包装是：

- 方法承诺：不是一张图，是一套可复制封面方法。
- 工具承诺：不只写文案，也做封面和图文卡片。
- 开源承诺：Skill 已开源，可装、可改、可共建。

推荐第一版封面方向：

```text
主标题：不只写文案
副标题：封面 / 图文 / 三平台内容包
角标：开源 Skill
底部：social-media-skills
```

备选更强版本：

```text
主标题：AI 社媒 Skill
副标题：小红书封面先跑通
角标：开源共建
底部：文案 / 封面 / 图文卡片
```

可借：

- 收藏驱动大于点赞驱动：封面要让人觉得以后能反复用。

可复用规则：

- 比例：1080 × 1440，3:4 竖版优先。
- 主标题：3-7 字最佳；知识类封面文字可占视觉主体。
- 层级：主标题、副标题、辅助标签三层足够。
- 位置：标题居中偏上或上方 1/3；底部别放关键信息。
- AI/科技类方向：极简信息图 + 真实截图，或深色科技系 + 荧光强调。
- 高互动封面类型：纯文字大字报、对比冲击型、主图+标题+副标题、抠图拼贴。
- 多图笔记：第 1 张决定点击；第 2-3 张放核心价值；最后一张放总结/互动。

## 四、taste 规则迁移

来自 `design-taste-frontend` 的封面化改写：

- 读场景：这不是 landing page，是 3:4 信息流封面。目标是 1 秒读懂，而不是展示设计能力。
- 三个刻度：设计变化 7-8，动效仅用于想象，视觉密度 4-6。
- 字体：默认中文无衬线，黑体体系；不要默认衬线，不要花哨渐变字。
- 颜色：一个强调色锁死全图；避免 AI 紫蓝渐变；不要纯黑纯白，改用炭黑/象牙白。
- 构图：不死居中；可用左大字 + 右卡片/截图，也可上标题 + 下截图。
- 信息：最多三层；标题 ≤ 10 字，副标题 ≤ 20 字。
- 禁忌：三等分卡片、装饰点满屏、假精确数据、emoji 堆砌、模板大渐变、空心描边字。

## 五、适合本项目的 5 个种子

### 种子 A：开源方法库

```text
主标题：不只写文案
副标题：封面 / 图文 / 三平台内容包
角标：开源 Skill
视觉：白底、黑粗字、红色角标、三张轻量平台卡片
```

### 种子 B：howto 社群语气

```text
主标题：howto 做社媒
副标题：小红书封面先跑通
角标：Skill 开源
视觉：白底 + 黑字 + 一块高亮色贴纸
```

### 种子 C：瑞士工具感

```text
主标题：社媒内容系统
副标题：公众号 / 小红书 / 抖音
角标：v0.1 不放封面，避免网页 taste 里的版本标签问题
视觉：网格、编号、平台标签、单一蓝色强调
```

### 种子 D：对比型

```text
主标题：别只写正文
副标题：先把封面做对
角标：小红书封面
视觉：左侧“正文堆字”灰掉，右侧“封面大字”高亮
```

### 种子 E：杂志感但不装

```text
主标题：图文自带骨架
副标题：封面 / 卡片 / 发布文案
角标：开源共建
视觉：轻杂志网格，不用衬线，不做复古纸张
```

## 六、推荐今天下午先做

先做 种子 A 与 种子 D 两张。

- 种子 A 负责介绍仓库能力。
- 种子 D 负责打痛点，适合传播。

两张都保持同一视觉系统：白/浅灰底、炭黑标题、红色强调、平台卡片。不要一张科技蓝、一张杂志黄，否则账号视觉不成体系。
