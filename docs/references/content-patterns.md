# 内容模式

## 通用输入

最少需要：

- 主题/素材。
- 目标平台。
- 目标受众。
- 人设口吻。
- 希望读者做什么：读完、收藏、评论、私信、转发、试用。

缺素材时，先用假设草稿，不伪装成真实经历。

## 公众号

结构：

1. 开头：冲突、问题、场景或反常识。
2. 主体：2-4 个论点，每个论点有证据或例子。
3. 转折：回答反对意见。
4. 结尾：一句可记住的判断 + 自然互动。

输出：

- 5 个标题：好奇心、反常识、利益点、故事感、方法论。
- 摘要：40-80 字。
- 正文。
- 头图方向：主体、文字、色调、比例。

## 小红书

结构：

1. 标题：关键词前置，利益/冲突明确。
2. 开头：一句说清“为什么要看”。
3. 正文：3-6 个点，每点短。
4. 结尾：低门槛互动。
5. 标签：主题、人群、场景、工具。

正文要像真实分享，不像产品说明书。可以口语化，但不要硬塞网络热词。

## 抖音

口播结构：

1. 前 3 秒：反常识/痛点/结果。
2. 解释：为什么。
3. 例子：一个具体场景。
4. 行动：让观众评论、保存或试。

写法：

- 每句可直接念。
- 关键句单独成行。
- 标出停顿、转折、画面提示。

## 标题策略

- 好奇心缺口：给结果，不给完整原因。
- 反常识：挑战默认认知。
- 痛点共鸣：说出读者没说出口的问题。
- 方法论：给明确收益和路径。
- 社交货币：让读者转发时显得有判断。
