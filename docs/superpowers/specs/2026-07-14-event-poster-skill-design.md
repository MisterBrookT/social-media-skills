# 活动海报 Skill 设计

## 目标

从立自π SOP 提取通用活动海报方法。新增小红书活动海报子能力；不固化立自π、MBTI、固定品牌色、固定 Logo、固定文案。

## 归属

- 顶层 owner：`xiaohongshu`。
- 新增：`skills/xiaohongshu/references/covers/event.md`。
- 路由：`skills/xiaohongshu/references/cover.md` 增加活动海报类型。
- 入口：`skills/xiaohongshu/SKILL.md` 增加活动海报读取条件。
- 不新增顶层 `event-poster`；保持平台顶层结构。

## 触发范围

- 单场活动招募。
- 系列活动预告。
- 活动详情长图。
- 活动回顾首图。

品牌介绍、产品介绍、普通方法/观点/资料封面继续走既有路由。

## 输入 contract

设计前收齐：

1. 海报类型。
2. 第一眼信息。
3. 用户关心理由。
4. 下一步动作。
5. 活动标题、价值主张。
6. 时间、地点、对象、流程、CTA。
7. Logo、图片、品牌色、禁用项。

关键事实缺失时标记待补；不编造时间、地点、名额、价格、报名方式。

## 工作流

1. 判断活动类型与传播目标。
2. 压缩四层信息：主标题、价值主张、解释、细节。
3. 选择单一视觉重心与明确对齐轴。
4. 建立颜色职责；主色不超过 2 个，辅助色不超过 2 个。
5. 完成首版；修改前先诊断文案、层级、排版、可读性、留白、颜色、装饰或素材问题。
6. 手机端与 3 秒阅读自检。

## 活动专属布局

- 招募：系列/品牌 → 活动标题 → 价值主张 → 时间地点对象 → 亮点/流程 → CTA。
- 预告：系列关系与场次差异优先；避免平均铺开。
- 详情长图：首屏完成标题、价值、关键事实；后续模块承载流程与适合对象。
- 回顾：真实现场素材为视觉证据；标题说明发生了什么，不伪造效果数据。

## 硬约束

- 主标题 3 秒内识别；手机信息流可读。
- 对齐轴明确；模块间距有规律。
- 主标题与副标题有明确视觉关系。
- 留白服务层级，不形成无意义空区。
- 装饰不压字；文字、Logo、素材保持对比。
- 避免会议议程、PPT 模板、信息表格、课程硬广感。
- 不把全部细节塞进首图；复杂信息转图文后续页或详情长图。

## 输出 contract

- 海报类型与传播目标。
- 四层文案。
- 版式说明：视觉重心、对齐轴、模块顺序。
- 颜色与素材职责。
- CTA。
- 发布前检查结果。

用户要求成图时，再调用可用设计或图像生成能力；本 reference 不绑定单一 harness 或工具。
